# Data Model: Dashboard ETL Pipeline

Defines the structure of the Delta tables dynamically generated by the streaming notebooks.

## Silver Layer: Multiple Typed Tables

Instead of multiplexing, the Silver stream filters the Bronze JSON blob into 4 distinct, schema-enforced Delta tables based on the incoming F1 `packet_id`.

### 1. `silver_events` (Packet 3)
| Column | Type | Origin | Notes |
|--------|------|--------|-------|
| `packet_id` | INT | Header | 3 |
| `car_index` | INT | Header | Player ID (0-19) |
| `timestamp` | TIMESTAMP | Header | - |
| `event_code` | STRING | Event | "SSTA", "RTMT", etc |

### 2. `silver_telemetry` (Packet 6 - 60Hz)
| Column | Type | Origin | Notes |
|--------|------|--------|-------|
| `packet_id` | INT | Header | 6 |
| `car_index` | INT | Header | Player ID (0-19) |
| `timestamp` | TIMESTAMP | Header | - |
| `speed_kmh` | FLOAT | Telemetry | Absolute |
| `engine_rpm_normalized`| FLOAT| Telemetry | Scaled 0.0 - 1.0 bounds |
| `gear` | INT | Telemetry | 0 = Neutral, 1-8 |
| `throttle` | FLOAT | Telemetry | 0.0 to 1.0 |
| `brake` | FLOAT | Telemetry | 0.0 to 1.0 |
| `engine_temperature` | INT | Telemetry | Absolute Celsius |

### 3. `silver_status` (Packet 7 - 2Hz)
| Column | Type | Origin | Notes |
|--------|------|--------|-------|
| `packet_id` | INT | Header | 7 |
| `car_index` | INT | Header | Player ID (0-19) |
| `timestamp` | TIMESTAMP | Header | - |
| `fuel_in_tank` | FLOAT | Status | |
| `ers_energy` | FLOAT | Status | |
| `tyre_compound` | STRING | Status | "C1", "C5" etc |
| `tyre_age_laps` | INT | Status | |
| `drs_activation`| BOOLEAN| Status | 0/1 boolean cast |

### 4. `silver_damage` (Packet 10 - 2Hz)
| Column | Type | Origin | Notes |
|--------|------|--------|-------|
| `packet_id` | INT | Header | 10 |
| `car_index` | INT | Header | Player ID (0-19) |
| `timestamp` | TIMESTAMP | Header | - |
| `front_wing_damage`| FLOAT | Damage | 0-100% |
| `rear_wing_damage` | FLOAT | Damage | 0-100% |
| `engine_wear` | FLOAT | Damage | 0-100% |
| `tyre_wear` | FLOAT | Damage | 0-100% (Overall wear map) |

## Gold Layer: Driver Distances (`gold_distances`)

Continuous stateful aggregation calculating where drivers are relative to each other dynamically.

| Column | Type | Notes |
|--------|------|-------|
| `window` | STRUCT | Contains `start` and `end` timestamps (1-second chunk) |
| `car_index` | INT | Primary key |
| `segment_distance` | FLOAT | Distance traveled specifically in this 1-second window |
| `avg_speed` | FLOAT | Mean speed across the window |

## Gold Layer: Vehicle Health (`gold_vehicle_health`)

Tracks the absolute latest state of vehicle degradation.

| Column | Type | Notes |
|--------|------|-------|
| `car_index` | INT | Primary Key |
| `last_updated` | TIMESTAMP | |
| `tyre_wear` | FLOAT | Captured from Damage Packet |
| `engine_wear` | FLOAT | Captured from Damage Packet |
| `front_wing_damage`| FLOAT | Captured from Damage Packet |
| `rear_wing_damage` | FLOAT | Captured from Damage Packet |
| `fuel_in_tank` | FLOAT | Captured from Status Packet |
| `ers_energy` | FLOAT | Captured from Status Packet |
| `drs_activation`| BOOLEAN | Captured from Status Packet |

## Gold Layer: Tyre Degradation (`gold_tyre_degradation`)

Cross-references telemetry distance traveled against the damage tyre wear percentage across specific compounds.

| Column | Type | Notes |
|--------|------|-------|
| `window` | STRUCT | Tumbling window boundary |
| `tyre_compound` | STRING | Aggregation grouping |
| `distance_traveled_m` | FLOAT | Total incremental distance traveled |
| `average_wear_pct` | FLOAT | Mean tyre wear over the measured distance |

## Gold Layer: Head-to-Head Gaps (`gold_driver_gaps`)

Provides live spatial distance gaps mathematically comparing trailing drivers to the race leader.

| Column | Type | Notes |
|--------|------|-------|
| `window` | STRUCT | Tumbling window boundary |
| `car_index` | INT | Primary driver ID |
| `gap_to_leader_m` | FLOAT | Cumulative spatial distance behind P1 |
