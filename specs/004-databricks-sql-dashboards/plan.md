# Implementation Plan: Databricks SQL Dashboards

**Branch**: `004-databricks-sql-dashboards` | **Date**: 2026-04-29 | **Spec**: [spec.md](./spec.md)

## Summary

Build three live Databricks SQL Dashboard widgets that read directly from the existing Gold Delta tables (`gold_driver_gaps`, `gold_vehicle_health`, `gold_tyre_degradation`) produced by the Medallion pipeline. No new pipeline code is required — this feature is purely presentation layer. Each widget maps to one user story and is independently deployable.

## Technical Context

**Language/Version**: Databricks SQL (ANSI SQL dialect)
**Primary Dependencies**: Databricks CLI (`databricks lakeview`), Gold Delta tables in `f1_catalog.gold`
**Deployment Method**: Fully CLI-driven — no manual UI clicks required. Dashboard defined as a `.lvdash.json` file and deployed via `databricks lakeview create` + `databricks lakeview publish`
**Storage**: Read-only from `f1_catalog.gold.*`; dashboard definition file stored in `src/databricks/dashboards/`
**Testing**: Manual validation — run synthetic generator, query Gold tables, verify widget output
**Target Platform**: Databricks Free Edition (formerly Community Edition) — includes Databricks SQL and AI/BI Dashboards
**Performance Goals**: Widget queries < 500ms; dashboard refresh every 30s
**Constraints**: Free Edition serverless-only; 30s refresh default to avoid competing with active streams
**Scale/Scope**: 20 cars × 4 widgets; queries are simple aggregations on small Gold tables

### [NEW] `src/databricks/dashboards/f1_dashboard.lvdash.json`

The complete AI/BI Dashboard definition file. Contains all datasets (SQL queries), widget layout, chart types, and refresh configuration in Databricks' `.lvdash.json` format. This single file is the source of truth for the entire dashboard.

**Deploy workflow (entire dashboard in 2 CLI commands)**:
```powershell
# 1. Create draft dashboard
databricks lakeview create --json @src/databricks/dashboards/f1_dashboard.lvdash.json
# → returns dashboard_id

# 2. Publish to make it accessible via shareable URL
databricks lakeview publish <dashboard_id>
```

**Update workflow (after adding/modifying widgets)**:
```powershell
databricks lakeview update <dashboard_id> --json @src/databricks/dashboards/f1_dashboard.lvdash.json
```

### [NEW] `src/databricks/dashboards/f1_dashboard_queries.sql`

All 4 widget SQL queries as standalone statements — useful for testing queries directly in the Databricks SQL Editor before embedding them in the `.lvdash.json` file.
